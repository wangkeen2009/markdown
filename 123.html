標題

若要建立標題，請在標題文字前加上一至六個 # 符號。 您使用的 # 數量將決定階層結構級別和標題的大小。

# 這是一級標題
## 這是二級標題
### 這是三級標題

使用兩個或多個標題時，GitHub 會自動產生一個目錄，可以透過點擊文件標頭中的圖示來存取。每個標題都列在目錄中，可以點擊某個標題導覽至所選部分。

文本樣式

您可以在評論欄位和 .md 檔案中以粗體、斜體、刪除線、下標或上標文字表示強調。

樣式 語法 鍵盤快捷鍵 範例 輸出
粗體 ** ** 或 __ __ Command + B (Mac) 或 Ctrl + B (Windows/Linux) **This is bold text** This is bold text
斜體 * * 或 _ _ Command + I (Mac) 或 Ctrl + I (Windows/Linux) _This text is italicized_ This text is italicized
刪除線 ~~ ~~ 無 ~~This was mistaken text~~ This was mistaken text
粗體和嵌入的斜體 ** ** 和 _ _ 無 **This text is _extremely_ important** This text is extremely important
全部粗體和斜體 *** *** 無 ***All this text is important*** All this text is important
下標 <sub> </sub> 無 This is a <sub>subscript</sub> text This is a subscript text
上標 <sup> </sup> 無 This is a <sup>superscript</sup> text This is a superscript text
底線 <ins> </ins> 無 This is an <ins>underlined</ins> text This is an underlined text

引用文本

您可以使用 > 來引用文本。

Text that is not a quote
> Text that is a quote

呈現效果：

Text that is a quote

注意： 查看對話時，可以透過突顯文字，然後鍵入 R 來自動引用評論中的文字。 可以透過依序點擊 和「引用回覆」來引用整條評論。

引用程式碼

使用單反引號可標註句子中的程式碼或命令。 反引號中的文字不會被格式化。 您也可以按 Command + E (Mac) 或 Ctrl + E (Windows/Linux) 鍵盤快捷方式將程式碼區塊的反引號插入到 Markdown 一行中。

Use `git status` to list all new or modified files that haven't yet been committed.

呈現效果：Use git status to list all new or modified files that haven't yet been committed.

要將程式碼或文字格式化為各自的不同區塊，請使用三反引號。

```
Some basic Git commands are:
git status
git add
git commit
```

呈現效果：

Some basic Git commands are:
git status
git add
git commit

有關詳細資訊，請參閱「建立和突顯程式碼區塊」。如果您經常編輯程式碼片段和表格，則可能受益於在 GitHub 上啟用對所有註解欄位採用固定寬度字體。 有關詳細資訊，請參閱「關於在 GitHub 上撰寫和設定格式」。

支援的顏色模型

在問題、拉取請求和討論中，可以使用反引號在句子中標註顏色。 反引號內支援的顏色模型將顯示顏色的視覺化效果。

The background color is `#ffffff` for light mode and `#000000` for dark mode.

呈現效果：The background color is #ffffff for light mode and #000000 for dark mode.

下面是目前支援的顏色模型。

顏色 語法 範例
HEX `#RRGGBB` `#0969DA` (#0969DA)
RGB `rgb(R,G,B)` `rgb(9, 105, 218)` (rgb(9, 105, 218))
HSL `hsl(H,S,L)` `hsl(212, 92%, 45%)` (hsl(212, 92%, 45%))

注意： 支援的顏色模型在反引號內不能有任何前導或尾隨空格。 顏色的視覺化效果僅在問題、拉取請求和討論中受支援。

連結

透過將連結文字用方括號 [ ] 括起來，然後將 URL 用括號 ( ) 括起來，可建立內嵌連結。 您也可以使用鍵盤快捷方式 Command + K 建立連結。 選取文字後，可以貼上剪貼簿中的 URL 以自動從所選內容建立連結。

This site was built using [GitHub Pages](https://pages.github.com/).

呈現效果：This site was built using GitHub Pages.

注意： 當評論中寫入了有效 URL 時，GitHub 會自動建立連結。

章節連結

您可以直接連結到具有標題的任何章節。 若要查看呈現檔案中自動產生的定位點，請將滑鼠懸停在章節標題上方以顯示圖示，然後點擊圖示以在瀏覽器中顯示定位點。

若需要確定要編輯的檔案中標題的定位點，可使用以下基本規則：

  • 字母轉換成小寫形式。
  • 空格由連字號 (-) 代替。
  • 任何其他空格或標點符號都將被刪除。
  • 前導和尾隨空格被刪除。
  • 標記格式被刪除，只保留內容 (例如，_italics_ 變為 italics)。
  • 如果標題的自動產生的定位點與同一文件中較早的定位點相同，那麼透過附加連字號和自動遞增整數來產生唯一識別碼。

相對連結

您可以在渲染的檔案中定義相對連結和圖片路徑，以協助讀者導覽至儲存庫中的其他檔案。

相對連結是相對於目前檔案的連結。 例如，如果在儲存庫根目錄下有一個 README 檔案，而在 docs/CONTRIBUTING.md 中有另一個檔案，則 README 檔案中的 CONTRIBUTING.md 的相關連結如下所示：

[Contribution guidelines for this project](docs/CONTRIBUTING.md)

GitHub 將根據您目前使用的分支自動轉換相對連結或圖片路徑，從而使連結或路徑始終有效。 連結的路徑將相對於目前檔案。 以 / 開頭的連結將相對於儲存庫根目錄。 可使用所有相對連結運算元，例如 ./ 和 ../。

自訂定位點

可使用標準 HTML 定位點標記 (<a name="unique-anchor-name"></a>) 為文件中的任何位置建立導覽定位點。 為了避免不明確的引用，請對定位點標記使用唯一的命名方案，例如向 name 屬性值新增前置詞。

可使用為定位點提供的 name 屬性的值連結到自訂定位點。 語法與連結到為標題自動產生的定位點時完全相同。

<h1>Section Heading</h1>
Some body text of this section.<a name="my-custom-anchor-point"></a>
Some text I want to provide a direct link to, but which doesn't have its own heading.
[A link to that custom anchor](#my-custom-anchor-point)

換行符

如果在儲存庫中的問題、拉取請求或討論中寫入內容，GitHub 將自動呈現換行符。

但是，如果在 .md 檔案中寫入，上面的範例將在一行中呈現，沒有換行符。 若要在 .md 檔案中建立換行符，需要包含以下內容之一：

  • 在第一行末尾包含兩個空格。 ( )
  • 在第一行末尾包含反斜線 (\)。
  • 在第一行末尾包含 HTML 單行分隔符標記 (<br/>)。

如果在兩行之間留出空白行，則問題、拉取請求和討論中的 .md 檔案和 Markdown 都將呈現由空白行分隔的兩行。

圖片

透過新增 ! 並將 alt 文字用 [ ] 括起來，可顯示圖片。 替換文字是等效於圖片中資訊的短文字。 然後將圖片的連結用括號 () 括起来。

![Octocat smiling and raising a tentacle.](https://myoctocat.com/assets/images/base-octocat.svg)

GitHub 支援將圖片嵌入到議題、拉取請求、討論、評論和 .md 檔案中。 您可以從儲存庫顯示圖片、新增線上圖片連結或上傳圖片。

注意： 想要顯示儲存庫中的圖片時，請使用相對連結而不是絕對連結。

HTML <picture> 元素也支援。

列表

可透過在一行或多行文字前面加上 -、* 或 + 來建立一個無序列表。

- George Washington
* John Adams
+ Thomas Jefferson

呈現效果：

  • George Washington
  • John Adams
  • Thomas Jefferson

要對列表排序，請在每行前面新增一個編號。

1. James Madison
2. James Monroe
3. John Quincy Adams

呈現效果：

  1. James Madison
  2. James Monroe
  3. John Quincy Adams

嵌套列表

透過在一個列表項下面縮排一個或多個其他列表項，可建立巢狀列表。

1. First list item
   - First nested list item
   - Second nested list item

呈現效果：

  1. First list item
    • First nested list item
    • Second nested list item

任務列表

若要建立任務列表，請在列表項前加連字號和空格，後接 [ ]。 要將任務標記為完成，請使用 [x]。

- [x] #739
- [ ] https://github.com/octo-org/octo-repo/issues/740
- [ ] Add delight to the experience when all tasks are complete :tada:

呈現效果：

  • #739
  • https://github.com/octo-org/octo-repo/issues/740
  • Add delight to the experience when all tasks are complete 🎉

如果任務列表項說明以括號開頭，則需要使用 \ 進行轉義：

- [ ] \(Optional) Open a followup issue

提及人員和團隊

您可以在 GitHub 上提及人員或團隊，方法是鍵入 @ 加上其使用者名稱或團隊名稱。 這將觸發通知並提請他們注意對話。

@github/support What do you think about these updates?

鍵入 @ 符號將顯示專案中的人員或團隊清單。 當您提及父團隊時，其子團隊的成員也會收到通知。

引用議題和拉取請求

可以透過鍵入 # 在儲存庫中調出建議的議題和拉取請求的清單。 鍵入議題或拉取請求的編號或標題以篩選清單，然後按 Tab 或 Enter 鍵以填寫選中的結果。

引用外部資源

如果自訂自動連結引用設定用於儲存庫，則對外部資源（如 JIRA 議題或 Zendesk 事件單）的引用將轉換為縮短的連結。

上傳資產

您可以透過拖放、從檔案瀏覽器中選取或貼上來上傳圖片等資產。 可以將資產上傳到議題、拉取請求、評論和儲存庫中的 .md 檔案。

使用表情符號

你可以透過鍵入 :EMOJICODE:（冒號後跟表情符號的名稱）將表情符號新增到寫作中。

@octocat :+1: This PR looks great - it's ready to merge! :shipit:

呈現效果：@octocat 👍 This PR looks great - it's ready to merge! 🚀

鍵入 : 將顯示建議的表情符號清單。

段落

透過在文字行之間留一個空白行，可建立新段落。

腳註

您可以使用此括號語法為您的內容新增腳註：

Here is a simple footnote[^1].
A footnote can also have multiple lines[^2].

[^1]: My reference.
[^2]: To add line breaks within a footnote, prefix new lines with 2 spaces.
  This is a second line.

呈現效果：

Here is a simple footnote1. A footnote can also have multiple lines2.

  1. My reference.↩
  2. To add line breaks within a footnote, prefix new lines with 2 spaces.
    This is a second line.↩

注意： Wiki 不支援腳註。

警報

警報是基於塊引用語法的 Markdown 擴展，可用于強調關鍵信息。 在 GitHub 上，它们以独特的颜色和图标显示，以指示内容的显著性。

要新增警報，請使用指定警報類型的特殊塊引用行，然後在標準塊引用中新增警報資訊。

Note

[!NOTE]
Useful information that users should know, even when skimming content.

Tip

[!TIP]
Helpful advice for doing things better or more easily.

Important

[!IMPORTANT]
Key information users need to know to achieve their goal.

Warning

[!WARNING]
Urgent info that needs immediate user attention to avoid problems.

Caution

[!CAUTION]
Advises about risks or negative outcomes of certain actions.

隱藏有評論的內容

你可以透過在 HTML 評論中加入內容來指示 GitHub 隱藏渲染的 Markdown 中的內容。

<!-- This content will not appear in the rendered Markdown -->

忽略 Markdown 格式

透過在 Markdown 字元前面輸入 \，可指示 GitHub 忽略 Markdown 格式（或對其進行轉義）。

Let's rename \*our-new-project\* to \*our-old-project\*.

呈現效果：Let's rename \*our-new-project\* to \*our-old-project\*.

注意： 在問題或拉取請求的標題中不會忽略 Markdown 格式。

禁用 Markdown 渲染

檢視 Markdown 檔案時，可點擊檔案頂部的「Code」以停用 Markdown 呈現並改為檢視檔案的來源。停用 Markdown 呈現使你能夠使用來源檢視功能，例如行連結，這在檢視呈現的 Markdown 檔案時不可用。
